## Architecture & Structure

**Directory Layout:** Flat `src/` with four modules (`config.rs`, `db.rs`, `mcp.rs`, `syslog.rs`) plus `main.rs`. Config file at root (`config.toml`). Containerized via `Dockerfile` and `docker-compose.yml`. Session docs under `docs/sessions/`.

**Entry Point:** `main.rs` — single `#[tokio::main]` async entry. Sequentially: loads config, initializes DB pool, starts syslog listeners, spawns a retention purge background task (hourly), builds the Axum HTTP router, and binds the MCP server with graceful shutdown via `ctrl_c`.

**Module Responsibilities:**
- `config.rs` — Configuration structs with defaults. Layered loading via Figment: hardcoded defaults, then `config.toml`, then env vars prefixed `SYSLOG_MCP_`.
- `syslog.rs` — Ingestion layer. UDP and TCP syslog receivers using Tokio. Parses RFC 3164/5424 via `syslog_loose`. Feeds parsed messages through `mpsc::channel(10_000)` to a batched writer.
- `db.rs` — Storage layer. SQLite via `r2d2` connection pool + `rusqlite`. Schema: `logs` table with B-tree indexes, FTS5 virtual table with porter stemmer synced via triggers, and a `hosts` summary table. Exposes `insert_log`, `insert_logs_batch`, `search_logs`, `tail_logs`, `get_error_summary`, `list_hosts`, `purge_old_logs`, `get_stats`.
- `mcp.rs` — MCP server implementing JSON-RPC 2.0 over HTTP. Three Axum routes: `POST /mcp`, `GET /sse`, `GET /health`. Dispatches MCP methods via a `match` statement.

**Patterns:**
- **Producer-consumer with channel buffering** — Syslog listeners produce `ParsedLog` structs into bounded `mpsc` channel (10k capacity). Single `batch_writer` consumer drains up to 100 messages or flushes every 500ms.
- **Shared state via `Arc`** — `r2d2::Pool` wrapped in `Arc`, shared between syslog ingestion and MCP query sides.
- **Manual JSON-RPC dispatch** — No MCP SDK. Hand-rolled `JsonRpcRequest`/`JsonRpcResponse` structs with a `dispatch` function routing method strings to handlers.
- **Dynamic SQL construction** — `append_filters` builds parameterized SQL clauses.
- **FTS5 with trigger-based sync** — Full-text search index maintained via SQLite triggers.

**MCP Protocol:** Implements MCP protocol version `2025-03-26`. Advertises `tools` capability only (no resources, no prompts).
